## Key Decisions (this session)
- **Full-fill threshold split at 1924:** spec said "skip feather/bg when gameBand ≥ 1916"; a hand-dragged game box TALLER than 9:16 (band > 1924) would visibly distort if force-scaled to 1920, so those scale true and crop the centered 1920 window instead. ≤ 4px of slop absorbs into the forced scale exactly as spec'd.
- **Chips also show when editing an already game-only layout** (not just fresh/nocam) — spec-completing: without it a saved game-only layout could never switch presets. Zero-cost, same visibility rule.
- **'nocam' does NOT auto-clear the cam box** — the chips are the explicit action; detection never silently rewrites the draft on that outcome.
- **No electron-store migration:** camRect null is a value-domain widening (old entries all carry valid rects; null is only produced by new code, and every reader now tolerates it). Old→new data needs no transform; fresh installs unaffected.

## Watch Out For
- **`{...maybeNullRect}` is the new whitelist trap variant** — spreading null gives `{}`, which downstream validators read as corrupt → silent no-reframe. Any NEW code copying camRect must null-guard (grep the four fixed sites for the pattern).
- **isReframeActive semantics:** camRect === null → active game-only; camRect undefined/corrupt → inactive. Don't "simplify" the two cases together.
- **#166 makes the calibration boxes invisible on first entry** until any resize — if a future CDP session drives box drags right after opening the editor and finds nothing, nudge a panel divider (ArrowLeft/Right on the resize handle) first; don't chase ghosts like this session did.
- **Old lessons standing:** every hidden `<video>` needs teardown; overlay renderer still on the deprecated positional `console-message` signature; don't re-run `npm run build` casually (dist/ alpha.5 installer is triply stale); detection determinism = compare rects with a few px tolerance, never equality.
