# ClipFlow UX Audit — Every Major User Path
_2026-07-01 (session 92). Method: 6 parallel per-view code auditors + independent verification of every headline claim in the main session + a deep-trace pass on Queue/publishing. Live-app pass pending (screen-control permission dialog timed out twice); the "Needs live eyes" list at the bottom is the remaining live script. Judged as a commercial product a stranger installs._

## Verdict

The app's **happy paths are in good shape** — progress feedback during generation is genuinely strong, the publish confirmation modal is a real safety gate, delete labeling in the editor timeline is clear, and the TikTok compliance panel is thorough. The consistent weakness is everything *off* the happy path: *when something fails, ClipFlow goes quiet.* And the first-run experience actively misleads a fresh install.

---

## The 6 themes

### 1. The app goes quiet exactly when things go wrong (biggest theme)
Success gets toasts, badges, and green checks. Failure gets `console.error` and nothing else. Confirmed silent-failure sites:
- Rename fails (locked file) → button appears to do nothing — [RenameView.js:609](src/renderer/views/RenameView.js:609), [main.js:565-575](src/main/main.js:565) returns raw `err.message`; the friendly locked-file pattern **already exists** at [main.js:1337-1354](src/main/main.js:1337) and just isn't reused
- Render fails in the editor → progress pill vanishes, no message — [EditorLayout.js:402-424](src/renderer/editor/components/EditorLayout.js:402)
- Clip preview can't play (file moved) → `.catch(() => {})` — [ProjectsView.js:295](src/renderer/views/ProjectsView.js:295)
- Queue: title save, platform toggle, dequeue, unschedule, caption save, TikTok field saves — all `console.error`-only; drag-reorder persistence is `.catch(() => {})` — [QueueView.js:662-749](src/renderer/views/QueueView.js:662), [QueueView.js:979](src/renderer/views/QueueView.js:979)
- IPC results shaped `{ error }` skip BOTH the optimistic update and the catch → silent no-op — pattern across QueueView handlers
- Batch apply in Rename's Manage tab: partial failure invisible — [RenameView.js:1273-1296](src/renderer/views/RenameView.js:1273)

**Pattern fix:** one shared error toast component + wire these ~10 catch sites to it. Small effort, transforms failure UX app-wide.

### 2. Destructive actions have three different safety levels, randomly assigned
- Project **single** delete: instant, no confirm, errors swallowed — [ProjectsView.js:893-896](src/renderer/views/ProjectsView.js:893), root handler [App.js:412-424](src/renderer/App.js:412)
- Project **bulk** delete: two-click confirm — [ProjectsView.js:880-891](src/renderer/views/ProjectsView.js:880)
- **Folder** delete: 5s undo toast — ProjectsView.js:939-963
- **Game/content-type** delete in Settings: instant — [SettingsView.js:285](src/renderer/views/SettingsView.js:285)
- **Account disconnect**: confirm modal — SettingsView.js:864-881
- **HIDE** a pending rename card: instant, no undo, loses in-progress edits — [RenameView.js:1024-1034](src/renderer/views/RenameView.js:1024)

**Pattern fix:** one rule — anything destructive gets confirm OR undo — applied to the three gaps (single project delete, game delete, HIDE).

### 3. A stranger's first launch lies to them (commercial blocker cluster)
- `watchFolder` defaults to **Fega's personal `W:\` path** on every fresh install — [App.js:123](src/renderer/App.js:123), [main.js:150](src/main/main.js:150)
- The Rename tab then shows a green pulsing **"WATCHING"** over that nonexistent folder — the watcher physically cannot report errors (`.on("add")`/`.on("unlink")` only, no `.on("error")`; `watcher:start` returns `{success:true}` unconditionally) — [main.js:669-697](src/main/main.js:669), RenameView.js:1350-1356
- Onboarding wizard covers AI style preferences only — never watch folder, FFmpeg, Whisper, or API keys (OnboardingView.js)
- Whisper setup asks a gaming creator to "browse to python.exe in BetterWhisperX venv" and says "ffmpeg must be installed and in PATH" — [SettingsView.js:938-952](src/renderer/views/SettingsView.js:938)
- No screen anywhere states the pipeline order (record → rename → generate → review → edit → queue → publish)
- No single-instance lock — two launched copies share one database — main.js (grep confirmed absent)
- Window size/position not remembered (hardcoded 1280×860)

### 4. Long waits are well-signaled; their failures aren't
- Generation progress is the app's best UX (10 named stages, live icons, signal health table, liveness pulse) — UploadView.js:1119-1239
- But: **no cancel** for a multi-minute run (only render has cancel); per-file failure reasons vanish after a 6s toast ([UploadView.js:361-382](src/renderer/views/UploadView.js:361)); quitting mid-run kills silently ([main.js:537-547](src/main/main.js:537)); nothing says clips landed in Projects afterward
- Publishing: partial success (2 of 3 platforms) shows the same red **"Failed"** as total failure — QueueView.js:1147; TikTok's 5-min poll timeout renders as "Failed" while its own message says "may still be processing" — tiktok-publish.js:312

### 5. Error message quality is platform-lottery
- TikTok has a real translation layer ([main.js:2520-2547](src/main/main.js:2520)) — its errors are the best in the app
- Instagram / Facebook / YouTube return raw `err.message` (SDK codes, HTTP bodies, `error_description` interpolation) straight to the UI and into the persistent Publish Log — main.js:2727/2778/2892, QueueView.js:1929
- "Please reconnect in Settings" errors give no button/link to Settings; an expired connection is invisible until a publish fails
- **Bug:** `facebook:publish` never checks token expiry at all — the other 3 platforms do (verified: `expiresAt` checks at main.js:2394/2444/2672/2842, none in the FB handler)

### 6. Keyboard users are locked out
- `outline: "none"` set inline on essentially every input app-wide with no focus style replacement (SettingsView.js:11 shared `inputStyle`, plus shared.js, modals.js, QueueView, TrackerView, RenameView, ProjectsView, OnboardingView…) — a keyboard user can't see where they are
- Primary interactions are `<div onClick>`: recording cards, queue rows, tracker cells, Manage-tab selection, TikTok compliance checkboxes, preview play/seek — none focusable
- Custom dropdowns open by keyboard but can't be operated by keyboard (options are `<div onClick>`, no arrow nav) — shared.js:105-139
- Editor is the bright spot: Space/S/Ctrl+Z/Del all work, tabs are real buttons; `TestChip` proves the team knows the accessible pattern (role/tabIndex/onKeyDown)
- Dense 8-10px text carries decision-critical info (queue status badges 9px, TEST chip 9px, tracker cells 8px). Density is a design value here — the fix is bumping only the *decision-carrying* text a notch, not de-densifying.

## Important findings

1. **No cancel for clip generation** (multi-minute AI run) — only quit-the-app stops it. UploadView.js generate flow.
2. **Partial publish reads as total failure** — "Failed" badge for 2-of-3 success; per-platform truth only visible expanded. QueueView.js:1147, 1213-1229.
3. **Raw API errors for IG/FB/YT** + no reconnect affordance from the error; connection expiry invisible until failure. main.js publish handlers.
4. **Facebook token never refreshed** (other 3 platforms refresh) — stale FB login fails with a raw Graph error. main.js:2733-2780.
5. **Autosave is invisible** — "last saved" only updates on manual save ([EditorLayout.js:224,255,272](src/renderer/editor/components/EditorLayout.js:224)); leaving the editor auto-saves without ever saying so.
6. **"Today" stat shows pending count**, not today's renames — [RenameView.js:1362](src/renderer/views/RenameView.js:1362).
7. **Batch-generate failure details unrecoverable** after the 6s toast; failed file shows no error badge. UploadView.js:361-382.
8. **Nothing routes the user Recordings → Projects** after generation ("Projects" is never mentioned); the ✓ badge is the only trace.
9. **Score "8.4/10", "High energy", "73% confidence" are never explained** — no tooltip anywhere. ProjectsView.js:488-504, 616-617.
10. **Keyboard focus invisible app-wide** (`outline:none` pattern) + primary rows/cards mouse-only (theme 6 above).
11. **No single-instance lock** — double-launch shares one SQLite DB. main.js.
12. **Missing/moved video file** → clip preview click silently does nothing. ProjectsView.js:279-302.
13. **Reschedule requires unschedule → re-schedule** — no direct "change time" on a scheduled clip. QueueView.js:1841-1897.
14. **Whisper/PATH/venv jargon** at the exact moment a non-technical customer is most fragile (setup). SettingsView.js:938-952.
15. **Processing project rows look clickable but aren't** (only opacity hints at it). ProjectsView.js:1237-1238.

## Suggestions (polish, when convenient)

- Yellow badge means three different things in Queue (Scheduled / Publishing… / Not rendered) — only 9px text disambiguates (QueueView.js:1219-1228); 12 AM sits last in the hour list; Sunday silently missing from date options; no timezone label on schedule times
- Empty project shows "No clips match this filter" — same copy as a filter miss (ProjectsView.js:1722); transcript hard-clips at 8 lines with no fade/"more" (ProjectsView.js:739); no full-title tooltip on truncated titles
- Segment-mode dropdown ("3 Words") has no field label (LeftPanelNew.js:182); context menu says "scene" where the row says "Audio" (TrackContextMenu.js:13 vs TimelinePanelNew.js:1014); Redo tooltip omits Ctrl+Shift+Z (EditorLayout.js:700)
- Publish confirm modal isn't Escape-dismissable; the placeholder-title warning at schedule-time uses a native `window.confirm` that matches nothing else in the app (QueueView.js:806); YouTube privacy dropdown is a native `<select>` with the exact dark-mode contrast problem the custom Select was built to avoid (QueueView.js:1576 vs 300-302)
- Settings save model is split (Edit→Save for text, instant for toggles) with no visual cue which is which; collapse state resets on relaunch
- "Clip" vs "Generate" used interchangeably for the same action (UploadView.js:1027/1082/1506); "SPLIT & RENAME" appears without the user having chosen a split (RenameView.js:1568)
- Dead weight: `src/main/publish.js` stubs (never called), unreachable `!clip.renderPath` guard (QueueView.js:1027), no-op `rebuildSegmentTags` (ThumbnailScrubber.js:138), inert Download button (LeftPanelNew.js:428-430 — no onClick), stale "export panel" wording in a TikTok error (tiktok-publish.js:358), renderer/main test-block wording mismatch ("project" vs "clip")
